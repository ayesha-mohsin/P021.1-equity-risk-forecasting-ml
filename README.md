# Equity Risk Forecasting & Portfolio Optimization
### ML-driven crash prediction, volatility forecasting, and risk-managed backtesting

---

## Overview

This project builds a full end-to-end machine learning pipeline for **forecasting equity market risk**. It predicts the probability of a market crash in the next 5 trading days, forecasts near-term volatility, and uses those predictions to run a walk-forward backtested portfolio strategy that significantly outperforms a buy-and-hold benchmark on a **risk-adjusted basis**.

The pipeline spans 17+ years of daily market data (2007–2024), covering two major crises (2008 GFC, 2020 COVID crash), and produces SHAP-based model explanations.

## Key Results

| Metric | ML Strategy | Buy & Hold (SPY) |
|---|---|---|
| Sharpe Ratio | **1.20** | 0.73 |
| Max Drawdown | **-11.72%** | -33.72% |
| Total Return | +392% | +496% |

> **Interpretation**: The strategy trades raw return for safety. A 64% better Sharpe ratio and 3× lower drawdown makes this the preferred profile for risk-averse investors (pension funds, endowments, family offices).

## Pipeline Phases

### Phase 1 — Data Collection (`src/data_loader.py`)
Downloads 17 years of daily market data from two free sources:
- **Yahoo Finance** (via `yfinance`): 8 ETFs — SPY, QQQ, IWM, TLT, HYG, GLD, EFA, EEM
- **Fama-French Data Library**: Market risk premium (Mkt-RF), SMB, HML, Risk-Free Rate
- **CBOE VIX**: Fear index (via Yahoo Finance ticker `^VIX`)

Output: `data/processed/merged.parquet` (~4,189 rows × multiple columns)

### Phase 2 — Feature Engineering (`src/features.py`)
Builds **27 predictive features** from raw prices and returns:

| Feature Group | Features | Rationale |
|---|---|---|
| Lag returns | 1d, 2d, 5d, 10d, 21d returns | Price momentum/reversal signals |
| Rolling stats | 5d/21d volatility, mean, skewness | Distributional regime changes |
| VIX features | Level, 5d/21d change, VIX/vol ratio | Market fear and dislocation |
| Fama-French | Mkt-RF, SMB, HML, RF | Factor risk exposures |
| 200-day MA signal | Price vs 200d MA (0/1) | Trend regime filter |
| Momentum divergence | Short vs long-term mom ratio | Momentum breakdown signal |
| Volatility regime | Ratio of 5d vol to 21d vol | Vol spike detection |
| Flight-to-safety | Rolling corr(SPY, TLT) | Risk-off indicator |
| **Peak drawdown** | Drawdown from 252-day high | **#1 most predictive feature** |

Output: `data/processed/features.parquet`

### Phase 3 — Model Training (`src/models.py`)
Trains three model types using **time-series cross-validation** (no data leakage):

**Regression** (predicting next-5-day realized volatility):
- Ridge Regression — RMSE: 0.0819 ✅ best
- Random Forest — RMSE: 0.0837
- XGBoost — RMSE: 0.0884

**Classification** (predicting crash: >2% loss in next 5 days):
- Logistic Regression — AUC: 0.675
- Random Forest — AUC: 0.682 ✅ best individual
- XGBoost — AUC: 0.632
- Calibrated Ensemble — AUC: 0.652

All experiments tracked in **MLflow** (`mlflow.db`). Models logged with parameters, metrics, and artifacts.

### Phase 4 — Statistical Testing (`src/evaluate.py`)
Validates that model predictions are statistically significant:
- **Paired t-tests**: Tests whether strategy returns are significantly different from benchmark
- **Confidence intervals**: Bootstrap CIs on Sharpe ratio and AUC
- **Fama-French regression**: Regresses strategy excess returns on market factors to check for alpha

### Phase 5 — Decision Layer (`src/decision.py`)
**Walk-forward backtesting** (the right way to test financial ML):
- Trains on rolling 3-year window, tests on next year — no future data leakage
- Uses Random Forest crash probability as the primary signal
- **Position sizing**: Scales equity exposure inversely to crash probability
- **Transaction costs**: 10bps per trade (realistic institutional assumption)
- **Benchmark**: SPY buy-and-hold

Key outputs: Sharpe 1.20 vs 0.73, MaxDD -11.72% vs -33.72%

### Phase 6 — Explainability (`src/explain.py`)
SHAP (SHapley Additive exPlanations) analysis on the best classification model:
- `shap_importance_bar.png` — Global feature importance ranking
- `shap_summary_dot.png` — Feature value vs impact distribution
- `shap_single_day.png` — What drove the prediction on the riskiest single day
- `shap_stability.png` — How feature importance varies over time

**Key findings**:
- Top feature: `peak_dd_252` (how far below the 252-day high the market currently is)
- Riskiest predicted day: **2008-10-02** (two weeks after Lehman Brothers collapsed)

```bash

pip install -r requirements.txt
```

```bash
python run_pipeline.py
```

```bash
python run_pipeline.py --skip-data
```

```bash
mlflow ui --backend-store-uri sqlite:///mlflow.db
# Open http://localhost:5000
```

## Design Decisions & Limitations

**Why walk-forward instead of simple train/test split?**
Financial data has temporal dependencies. Using a simple 80/20 split would allow the model to "see" future market regimes during training. Walk-forward simulation respects the arrow of time.

**Why is the strategy return lower than buy-and-hold (+392% vs +496%)?**
The model correctly reduces equity exposure during high-risk periods — including the 2009 and 2020 bull market recoveries. It misses some upside to avoid catastrophic drawdowns. This is the intended tradeoff for a risk-management strategy.

**Why AUC ~0.68 and not higher?**
Markets are notoriously difficult to predict. An AUC of 0.68 on a daily crash prediction task is considered strong in academic finance literature. The key is that even modest predictive power, when combined with proper position sizing and risk management, produces significant portfolio improvement.

**What's in `.gitignore`?**
Raw downloaded data files, large model artifacts, and `__pycache__` directories. The `mlflow.db` is tracked to preserve experiment history.
